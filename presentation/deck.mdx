# SSG vs SSR

- Ambos tienen sus pros y contras.
- SSG Genera ahead of time pero puede resultar caro reconstruir todo el sitio en cada cambio.
- SSG Es mas intenso en recursos en sitios grandes (Martillo/Pica).
- SSR Permite controlar status codes (Redirects, Not-Found, etc).
- SSR Necesita infraestructura de por medio.
- SSR es mas apto para manejo de cookies y escenarios complejos.

Y prerender?

- Prerender es la opcion mas barata para muchisimos escenarios y luce mas como SSG.

# Optimizaciones baratas para apps modernas

- robots.txt - Es un archivo que permite decirle a los crawlers que contenido no deseas indexar, util para deduplicar.
- sitemap.xml - Es un archivo que le sugiere a un web crawler de que manera traversar tu sitio, puede agilizar el proceso.
- Manejo de status codes - Importantisimo!
- URL Cortas - Al igual que las clases en el codigo, las url largas confunden a las personas y a los bots.
- Usa rel="canonical" para deduplicar contenido.
